SEO is one of the main acquisition channels for Roofwander.

But marketplaces have a particular challenge.

New pages appear constantly: listings, guides, blog posts, landing pages.

Search engines need to discover them quickly, but also understand how everything connects.

So we built a simple SEO infrastructure around that.

Sitemap architecture

Instead of one sitemap, we use a sitemap index separating content types:

• general pages
• recent listings
• blog posts
• guides

This helps search engines understand the structure of the site and crawl the most relevant sections.

We also use locale-specific URLs to support international SEO.

Automatic discovery for listings

Marketplace content changes constantly.

New listings are automatically added to sitemaps so crawlers can discover them without manual submission.

We also implemented push-based discovery:

• IndexNow for Bing
• Google Indexing API

Instead of waiting for crawlers, we notify search engines when new pages appear.

Helping search engines understand the site

Discovery is only part of SEO.

Search engines also need to understand how pages relate to each other.

So we structure internal links across the platform:

• guides linking to listings
• blog articles linking to marketplace pages
• location and brand pages connecting related listings

This helps both users and search engines navigate the ecosystem.

These decisions helped us structure SEO for a platform where new content is generated continuously.
