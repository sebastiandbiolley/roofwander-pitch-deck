One thing that helped us a lot at Roofwander: using HTML for business documents.

For partnership one-pagers, contracts, and even invoice examples, we did not want to depend only on Word files or constantly redesign things manually.

So we started structuring these documents as HTML files.

Why?

• cleaner and more consistent layouts
• easy to version with Git
• easy to duplicate and translate
• easy to reuse across partners and document types
• simple to export as PDF from the browser

What I like is that it sits somewhere between design and ops:

not a heavy design workflow,
not messy document editing either.

It makes it easier to maintain a small internal document system for things like:

• brand partnership one-pagers
• reseller documents
• contract templates
• invoice examples
